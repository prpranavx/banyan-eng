===========================================
 FRONTEND REQUIREMENTS (/frontend)
===========================================

Use:
- React + Vite
- Clerk authentication
- TailwindCSS

Include these pages/components:
1. Login page (Clerk <SignIn />)
2. Dashboard (only accessible after login)
   - Button: “New Interview”
   - Shows a generated interview session link (frontend URL + /candidate/:id)
   - Table: Active interviews
   - Table: Past interviews

3. Candidate Interview Page
   - Super minimal UI: a chat box, message list
   - Sends messages to backend (/api/send-message)
   - Shows AI responses
   - Does NOT require login (candidates do not authenticate)

Environment variables:
- VITE_CLERK_PUBLISHABLE_KEY
- VITE_BACKEND_URL

Routing:
- / (dashboard — protected)
- /sign-in
- /candidate/:sessionId

===========================================
 BACKEND REQUIREMENTS (/backend)
===========================================

Use:
- Express + TypeScript
- CORS
- OpenAI API (new “Responses” API)

Endpoints:
1. POST /api/generate-session
   - Creates a sessionId (UUID)
   - Returns { sessionId }

2. POST /api/send-message
   - Input: { sessionId, message, codeSnapshot? }
   - Processes via OpenAI
   - Returns AI assistant message

3. POST /api/evaluate
   - Input: { sessionId }
   - Summarizes candidate performance
   - Returns scoring JSON

Middleware:
- Clerk JWT verification for interviewer-only routes
- Candidates do NOT need Clerk login

Environment variables:
- OPENAI_API_KEY
- CLERK_SECRET_KEY
- PORT

===========================================
 WORKER REQUIREMENTS (/worker)
===========================================

Use:
- Playwright (Chromium)
- TypeScript

The worker should:
1. Poll the backend for new sessions to attach to (stub API or placeholder).
2. When a new session appears:
   - Launch Chromium
   - Navigate to a HARD-CODED placeholder URL (e.g. https://coderpad.io)
   - Inject a small floating chat box into the DOM
   - Capture candidate code editor content every 2 seconds
   - POST updates to backend (/api/send-message)

3. Listen for AI replies from the backend and insert them into the chat box.

4. Keep everything minimal — this is an MVP placeholder worker.

Environment variables:
- BACKEND_URL
- PLAYWRIGHT_BYPASS_CSP=true
